## Consigna

Cite otras características importantes de Python, Ruby, PHP, Gobstone y Processing.
Por ejemplo: tipado de datos, cómo se organizan los programas, etc.


## Respuesta

#### Python

- Tipado dinámico: el tipo de las variables se determina en tiempo de ejecución.
- Tipado fuerte: no permite conversiones implícitas inseguras entre tipos.
- Interpretado: no requiere compilación previa.
- Gestión automática de memoria (garbage collector).
- Multiplataforma (funciona en distintos sistemas operativos).
- Gran ecosistema para ciencia de datos, IA y automatización.
- Soporte para programación asincrónica (async/await).
- Permite scripting rápido y prototipado.
- Alto nivel de productividad del desarrollador.

#### Ruby

- Tipado dinámico y fuerte.
- Lenguaje completamente orientado a objetos (todo es un objeto).
- Interpretado.
- Sintaxis flexible y expresiva, cercana al lenguaje natural.
- Metaprogramación (permite modificar clases y comportamiento en tiempo de ejecución).
- Filosofía centrada en la simplicidad y productividad del programador.
- Muy utilizado con el framework Ruby on Rails.
- Soporte para bloques, procs y lambdas.
- Alta expresividad del código.

#### PHP

- Tipado dinámico.
- Lenguaje interpretado del lado del servidor.
- Diseñado principalmente para el desarrollo web.
- Amplio soporte para bases de datos (MySQL, PostgreSQL, etc.).
- Gran disponibilidad de hosting web compatible.
- Frameworks populares como Laravel o Symfony.
- Evolución hacia mejor soporte de programación orientada a objetos.
- Manejo integrado de sesiones y cookies.
- Muy utilizado en sistemas web existentes (gran adopción).

#### Gobstone

- Lenguaje educativo orientado a la enseñanza de programación.
- Paradigma imperativo.
- Tipado simple.
- Entorno controlado que facilita el aprendizaje sin errores complejos.
- Permite visualizar fácilmente el estado del programa.
- Ideal para enseñar lógica y algoritmos básicos.
- Reduce la complejidad sintáctica para enfocarse en la lógica.

#### Processing

- Lenguaje basado en Java.
- Diseñado para gráficos, animaciones e interacción visual.
- Paradigma imperativo y orientado a objetos.
- Entorno de desarrollo propio (IDE sencillo e integrado).
- Soporte para animación en tiempo real.
- Integración con hardware (como Arduino).
- Comunidad enfocada en arte generativo y visualización.
